<task type="auto">
  <name>Task 2: Create standalone verification script for Phase 1 requirements</name>
  <files>scripts/verify_phase1.py</files>
  <action>
Create `scripts/verify_phase1.py` -- a standalone, re-runnable verification script. The script must be independently runnable via `python scripts/verify_phase1.py` (no pytest dependency).

**Script structure:**

1. **Header/environment info:** Print verification date (datetime.now().isoformat()), Python version (sys.version), OS (platform.platform()).

2. **Synthetic data generator** `_make_synthetic_csv(tmp_dir)`:
   - Generate a DataFrame with column names matching `_DOMAIN_SHORTLIST` from preprocess.py (all 44 features). Reference the constant directly or copy the list.
   - Use at least 200 rows with numpy RandomState(42) for determinism.
   - Include a ` Label` column (leading whitespace, matching CICIDS2017 format) with mix of "BENIGN" (120) and "DDoS" (80).
   - Inject `np.inf` and `np.nan` into `Flow Bytes/s` column (rows 0 and 1) to verify cleaning.
   - Add identifier columns that should be dropped: `Flow ID`, `Source IP`, `Destination IP`, `Source Port`, `Destination Port`, `Timestamp`.
   - Save as CSV to tmp_dir, return path.

3. **Real data availability check** `_check_real_data_available(config)`:
   - Load config via `load_config("config/default.yaml")`.
   - Check if configured CSV files exist at configured raw_dir path.
   - Return boolean.

4. **Check functions** (one per requirement, each returns a dict with keys: req_id, check, status, value, method):

   - `check_data_01(synthetic_csv_path, real_data_available, config)`:
     - DATA-01: Load CSV via `load_cicids2017()` with a temp config pointing to synthetic CSV.
     - Verify zero Inf/NaN in result numeric columns via `np.isinf` and `np.isnan`.
     - If real data available, also run on real data and capture row count.
     - Method: "load_cicids2017() + np.isinf/isnan check"

   - `check_data_02(df_cleaned, config)`:
     - DATA-02: Call `select_features(df, config)` on the cleaned DataFrame.
     - Verify returned DataFrame has between 20-40 feature columns.
     - Also verify `select_features` is importable from `federated_ids.data` (the re-export from Task 1).
     - Method: "select_features() + len(columns) range check"

   - `check_data_03(df_cleaned, config)`:
     - DATA-03: Call `preprocess(df, config)` to get train/test splits with scaler applied.
     - Verify train feature means are near 0 (atol=1e-6 via np.allclose).
     - Verify test feature means are NOT all near 0 (not np.allclose with atol=0.05), confirming scaler was fitted on train only.
     - Use at least some features (check that at least 50% of test feature means deviate from 0, rather than requiring ALL, to avoid false positives with small synthetic data per Pitfall 2 in research).
     - Method: "preprocess() + np.allclose on feature means"

   - `check_data_05(preprocess_result, config)`:
     - DATA-05: Call `partition_iid()` with num_clients from config (test with 3 clients).
     - For each partition, compute class ratio (DDoS fraction).
     - Verify all partitions' class ratios are within 5% of global class ratio.
     - Method: "partition_iid() + class ratio deviation check"

   - `check_infr_01()`:
     - INFR-01: Call `load_config("config/default.yaml")`.
     - Verify presence of 5 required hyperparameters: training.learning_rate, training.local_epochs, training.batch_size, federation.num_rounds, federation.num_clients.
     - Report actual values found.
     - Method: "load_config() + key presence check"

5. **Summary table printer** `print_summary_table(results)`:
   - Print formatted table matching CONTEXT.md format: `| REQ ID | Check | Method | Result | Value |`
   - Print a final line: "X/5 checks passed"

6. **main():**
   - Print environment header.
   - Create temp directory for synthetic data.
   - Generate synthetic CSV.
   - Check real data availability.
   - Run all 5 checks in order.
   - Print summary table.
   - If real data was available, print a separate "Real Data Results" section.
   - Clean up temp directory.
   - Exit with code 0 if all pass, 1 if any fail.

**Important constraints:**
- Do NOT invoke pytest from the script.
- Do NOT catch/suppress exceptions in check functions -- let failures propagate as FAIL.
- Use `tempfile.mkdtemp()` for synthetic data workspace, clean up with `shutil.rmtree`.
- The script generates its own synthetic data inline -- does NOT depend on conftest.py fixtures.
- Handle missing real data gracefully: mark real data sections as "SKIPPED -- data not available".

**Pitfall avoidance:**
- Synthetic data must include all 44 domain shortlist column names so select_features works correctly (Pitfall 1).
- For DATA-03 scaler leakage check, verify that at least 50% of test feature means deviate from 0, not ALL (Pitfall 2).
- Read real data paths from config/default.yaml via load_config(), do not hardcode paths (Pitfall 3).
