  <action>
    Modify `src/federated_ids/fl/server.py`:

    1. Add conditional import at module top (after existing imports):
       ```python
       try:
           from torch.utils.tensorboard import SummaryWriter
           _HAS_TENSORBOARD = True
       except ImportError:
           _HAS_TENSORBOARD = False
       ```

    2. Inside `run_federated_training()`, BEFORE the FL loop (after `checkpoint_path` setup):
       ```python
       tb_dir = os.path.join(output_dir, "tensorboard")
       writer = SummaryWriter(log_dir=tb_dir) if _HAS_TENSORBOARD else None
       ```

    3. Wrap the FL loop and post-training code in try/finally:
       ```python
       try:
           for round_num in range(1, num_rounds + 1):
               # ... existing code ...
               metrics = server_evaluate(...)

               # TensorBoard logging (after existing per-round log line)
               if writer is not None:
                   writer.add_scalar("Global/loss", metrics["loss"], round_num)
                   writer.add_scalar("Global/accuracy", metrics["accuracy"], round_num)
                   writer.add_scalar("Global/f1", metrics["f1"], round_num)
                   writer.add_scalar("Global/precision", metrics["precision"], round_num)
                   writer.add_scalar("Global/recall", metrics["recall"], round_num)

               # ... existing checkpoint code ...
               history.append(metrics)
       finally:
           if writer is not None:
               writer.close()
       ```

    4. Keep ALL existing functionality unchanged -- this is purely additive. The try/finally wraps the entire loop + post-training section so writer.close() is guaranteed.

    5. Add a log line when TB is active: `logger.info("TensorBoard logging enabled: %s", tb_dir)` after creating the writer.
