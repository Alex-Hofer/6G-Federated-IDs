  <action>
1. Add the following new tests to `tests/test_fl.py` (RED phase):
   - `test_config_driven_rounds_clients`: Uses synthetic data (from conftest fixtures). Create a minimal config dict with `federation.num_rounds=2`, `federation.num_clients=2`, call `run_federated_training(config, ...)` or a simplified integration helper. Assert `len(history) == 2`.
   - `test_convergence_check_pass/fail`: Call `check_convergence` with crafted history lists.
   - `test_metrics_json_output`: Call `save_fl_metrics` with a sample history and config, assert file exists and JSON structure is correct.
   - `test_round_metrics_keys`: Assert each dict in a history list returned from a mini FL run has all 5 metric keys.
   - `test_config_banner`: Call `print_config_banner` with sample config, assert no exception.

2. Extend `src/federated_ids/fl/server.py` with these functions (GREEN phase):

   a. `print_config_banner(config: dict, device: str, num_features: int) -> None`:
      - Print ASCII box with aligned key-value pairs: Clients, Rounds, Local epochs, Batch size, Learning rate, Model architecture, Dropout, Features, Device, Strategy (FedAvg + fraction_fit), Seed.
      - Use `logger.info()` for each line.

   b. `check_convergence(history: list[dict], n: int = 3) -> bool`:
      - If `len(history) < 2*n`, adjust `n = max(1, len(history) // 2)`.
      - Compute `early_f1 = mean([h["f1"] for h in history[:n]])`, `late_f1 = mean([h["f1"] for h in history[-n:]])`.
      - Log early/late F1 and PASS/FAIL.
      - Return `late_f1 > early_f1`.

   c. `save_fl_metrics(history: list[dict], config: dict, output_path: str) -> None`:
      - Build payload dict with `"config"` (num_clients, num_rounds, local_epochs, batch_size, learning_rate, model hidden_layers, strategy="FedAvg", fraction_fit, device, seed) and `"rounds"` (list of `{"round": i+1, **metrics}`).
      - `os.makedirs(os.path.dirname(output_path), exist_ok=True)`, write JSON with `indent=2`.

   d. `run_federated_training(config: dict, config_path: str | None = None) -> list[dict]`:
      - **Setup:** Extract federation, training, model, data config sections. Call `set_global_seed(config.get("seed", 42))`, `device = get_device()`.
      - **Logging:** Set up logging with `basicConfig(level=config.get("log_level", "INFO"), format="%(asctime)s %(name)s %(levelname)s %(message)s")`. Suppress client-level training logger: `logging.getLogger("federated_ids.model.train").setLevel(logging.WARNING)`.
      - **Data pipeline auto-run:** If `not _cache_exists(processed_dir)`, import and call `main(config_path)` from `federated_ids.data.preprocess`.
      - **Load tensors:** Load X_train.pt, y_train.pt, X_test.pt, y_test.pt from processed_dir using `torch.load(..., weights_only=True).numpy()`.
      - **Partition:** Call `partition_iid(X_train, y_train, num_clients, seed)` then `create_dataloaders(partitions, X_test, y_test, batch_size)` to get `client_loaders` and `test_loader`.
      - **Class-weighted loss:** Load `class_weights.json` if `training.weighted_loss` is True, build `CrossEntropyLoss(weight=weight_tensor.to(device))`. Otherwise plain `CrossEntropyLoss()`.
      - **Initial model:** Create `MLP(input_dim, hidden_layers, num_classes, dropout).to(device)`.
      - **Print config banner** with num_features = input_dim.
      - **Get initial global parameters** via `get_parameters(model)` helper (same logic as FederatedClient.get_parameters but standalone).
      - **FL Loop:** For `round_num` in `range(1, num_rounds + 1)`:
        - `round_results = []`
        - For each `client_id` in range(num_clients): Create fresh `MLP` on device, create fresh `Adam` optimizer (reset optimizer state per round per McMahan FedAvg paper), create `FederatedClient(model, client_loaders[client_id], criterion, optimizer, device)`. Call `params, n_examples, _ = client.fit(global_params, {"local_epochs": local_epochs})`. Append `(params, n_examples)` to round_results.
        - `global_params = fedavg_aggregate(round_results)`
        - `metrics = server_evaluate(global_params, eval_model, test_loader, criterion, device)` where `eval_model` is a shared model instance for evaluation.
        - Log one-line: `Round %2d/%d -- loss: %.3f, acc: %.2f, F1: %.2f, prec: %.2f, rec: %.2f`
        - Checkpoint: If `metrics["f1"] > best_f1`, save `state_dict` to `outputs/checkpoints/global_model.pt`, log `Saved global model (round N, F1=X.XX)`.
        - Append metrics to history.
      - **Post-training:** Print summary table (reuse `_print_summary_table` pattern from Phase 2 train.py but adapted for "Round" column header instead of "Epoch"). Call `check_convergence(history)`. Call `save_fl_metrics(history, config, metrics_path)`. Log metrics file path.
      - Return history.

   NOTE: For the orchestration loop, create a fresh model per client per round to avoid shared model state (Pitfall 6 from research). Use the same model architecture for all clients. Reset optimizer state each round (Pitfall 2).

3. Update `src/federated_ids/fl/__init__.py` to export the new public functions: `run_federated_training`, `check_convergence`, `save_fl_metrics`, `print_config_banner`.

4. Run `pytest tests/test_fl.py -x -v` -- all tests MUST pass (GREEN).
5. Run `pytest tests/ -v` -- full suite must stay green.
