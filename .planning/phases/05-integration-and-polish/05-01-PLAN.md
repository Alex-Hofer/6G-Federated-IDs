<task type="auto">
  <name>Task 1: Create pipeline runner module and register console script</name>
  <files>src/federated_ids/pipeline.py, pyproject.toml</files>
  <action>
Create `src/federated_ids/pipeline.py` -- a thin orchestration module (~70-90 lines) that chains existing entry points sequentially.

**`run_pipeline(config_path: str = "config/default.yaml") -> None`:**
1. Call `load_config(config_path)` to get config dict
2. Call `set_global_seed(config.get("seed", 42))` for reproducibility (INFR-02)
3. Call `logging.basicConfig()` ONCE at the start (level from `config.get("log_level", "INFO")`, format: `"%(asctime)s %(name)s %(levelname)s %(message)s"`)
4. Stage 1 -- Preprocessing: Check `_cache_exists(processed_dir)`. If cached, log skip message. If not, import and call `preprocess.main(config_path)`. Use `processed_dir = config["data"].get("processed_dir", "./data/processed")`
5. Stage 2 -- FL Training: Import and call `run_federated_training(config, config_path=config_path)`. This already handles its own checkpoint saving (MODL-03).
6. Stage 3 -- Evaluation: Import and call `eval.__main__.main(config_path)`
7. Call `_print_pipeline_summary(output_dir)` at the end
8. Each stage wrapped in a log message: `"=== Stage N/3: {Name} ==="`
9. NO try/except around stages -- let exceptions propagate (fail fast per user decision). The stage functions themselves handle their own error conditions.

CRITICAL: Always pass `config_path` explicitly to every stage call. Never pass `None`. This avoids argparse conflicts (Pitfall 1 from research).

**`_print_pipeline_summary(output_dir: str) -> None`:**
1. Walk `output_dir` using `os.walk()`, collect all files
2. For each file, print path (relative to output_dir) and size in human-readable format (bytes/KB/MB)
3. Parse `{output_dir}/metrics/fl_metrics.json` -- extract the last round's metrics
4. Print key metrics: F1, precision, recall, accuracy from the final round
5. Print a separator line and "Pipeline complete." message

**`main() -> None`:**
1. Create argparse parser with `--config` flag (default: `config/default.yaml`), consistent with all other entry points
2. Call `run_pipeline(args.config)`

**Update `pyproject.toml`:**
Add one line to `[project.scripts]`:
```
federated-ids-run-all = "federated_ids.pipeline:main"
```

Use lazy imports (import inside function body) for the stage modules to keep the module import lightweight, consistent with the pattern in fl/server.py lines 284-322.

Do NOT modify any existing Phase 1-4 source files. The pipeline runner uses existing public APIs only.

     files: [synthetic.csv]

     target_features: 30
     correlation_threshold: 0.95
